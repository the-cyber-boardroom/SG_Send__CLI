### RF-1: Create Missing Safe_* Types (CRITICAL)

Create new domain-specific Safe_* types to replace raw primitives.

#### RF-1.1: Safe_UInt__Key_Size
```python
# sg_send_cli/safe_types/Safe_UInt__Key_Size.py
from osbot_utils.type_safe.primitives.core.Safe_UInt import Safe_UInt

class Safe_UInt__Key_Size(Safe_UInt):
    min_value = 256
    max_value = 16384
```

#### RF-1.2: Safe_Str__PEM_Key
```python
# sg_send_cli/safe_types/Safe_Str__PEM_Key.py
import re
from osbot_utils.type_safe.primitives.core.Safe_Str import Safe_Str

class Safe_Str__PEM_Key(Safe_Str):
    regex       = re.compile(r'[^\x20-\x7E\n\r]')
    max_length  = 8192
    allow_empty = True
```

#### RF-1.3: Safe_Str__Content_Type
```python
# sg_send_cli/safe_types/Safe_Str__Content_Type.py
import re
from osbot_utils.type_safe.primitives.core.Safe_Str import Safe_Str

class Safe_Str__Content_Type(Safe_Str):
    regex       = re.compile(r'[^a-zA-Z0-9\-\.+/]')
    max_length  = 128
    allow_empty = True
```

**Note:** These new types need companion tests — this is a Phase Two exception where we add tests for new types only.

**Assessment:** High effort, high value. Should be its own dedicated phase after PT-1 through PT-3.

**From brief:** "Try to make code changes that alter the application's behaviour without any test failing."

This exercise runs during Phase 3, after refactoring is complete:

| Adversarial Change | Expected Result |
|--------------------|-----------------|
| Change PBKDF2 iterations from 600K to 1 | Test must fail (interop vectors) |
| Change AES key size from 32 to 16 bytes | Test must fail (crypto tests) |
| Change HKDF info prefix | Test must fail (interop vectors) |
| Remove vault_key format validation | Test must fail (parse tests) |
| Change object ID length from 12 to 8 | Test must fail (ID format tests) |
| Remove legacy vault guard | Test must fail (guard tests) |
| Change default base URL | Test must fail (API setup tests) |
| Skip file encryption in push | Test must fail (roundtrip tests) |
| Change tree JSON structure | Test must fail (tree tests) |
| Remove signature verification | Test must fail (PKI tests) |

Every undetected change = write the missing test. Then revert the adversarial change.

**Goal:** Reach a state where any code change that changes behaviour triggers at least one test failure.
