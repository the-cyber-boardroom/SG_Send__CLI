## Test Performance Requirements

- All new unit tests must complete in < 100ms each
- Use `setUpClass` for expensive setup (crypto key generation, temp dirs)
- Share fixtures across test methods where possible
- No `time.sleep()` in unit tests
- Integration tests (WP-8) may use local server — cap at 5s per test

## Bug & Vulnerability Discovery Protocol

**From brief:** "When a bug is found, write a test that passes with the current (buggy) behaviour. Same for vulnerabilities."

### Bugs
When a test reveals unexpected behavior:
1. Document in `team/villager/qa/bugs/BUG-NNN__description.md`
2. Write a **passing** test that captures the current (buggy) behavior
3. Add a comment: `# BUG-NNN: <description of expected vs actual>`
4. Do NOT fix the bug — that's Phase 3 work
5. Add to the Phase 3 backlog

### Vulnerabilities
When a security issue is found:
1. Document in `team/villager/appsec/vulns/VULN-NNN__description.md`
2. Write a **passing** test that captures the current (vulnerable) behavior
3. Add a comment: `# VULN-NNN: <description of the vulnerability>`
4. Do NOT fix the vulnerability — that's Phase 3 work
5. Add to the Phase 3 backlog

The test suite is always green. Bugs and vulnerabilities are tracked as test assertions.
