## Data Flow: Clone Operation

```mermaid
sequenceDiagram
    participant User
    participant CLI as CLI__Vault
    participant Sync as Vault__Sync
    participant API as Vault__API
    participant Crypto as Vault__Crypto
    participant Store as Object_Store
    participant Refs as Ref_Manager

    User->>CLI: sg-send clone <vault_key>
    CLI->>Sync: clone(vault_key, directory)
    Sync->>Crypto: parse_vault_key(vault_key)
    Crypto-->>Sync: {vault_id, passphrase}
    Sync->>Crypto: derive_keys(passphrase, vault_id)
    Crypto-->>Sync: {read_key, write_key, tree_file_id, settings_file_id, ref_file_id}

    Sync->>API: read(vault_id, ref_file_id)
    API-->>Sync: encrypted_ref
    Sync->>Crypto: decrypt(read_key, encrypted_ref)
    Crypto-->>Sync: ref_json {commit_id, version}

    Sync->>API: read(vault_id, settings_file_id)
    API-->>Sync: encrypted_settings
    Sync->>Crypto: decrypt(read_key, encrypted_settings)
    Crypto-->>Sync: settings_json

    Sync->>API: read(vault_id, tree_file_id)
    API-->>Sync: encrypted_tree
    Sync->>Crypto: decrypt(read_key, encrypted_tree)
    Crypto-->>Sync: tree_json {files: [{path, file_id}...]}

    loop For each file in tree
        Sync->>API: read(vault_id, file_id)
        API-->>Sync: encrypted_file
        Sync->>Store: store(encrypted_file)
        Store-->>Sync: object_id
        Sync->>Crypto: decrypt(read_key, encrypted_file)
        Crypto-->>Sync: plaintext
        Sync->>Sync: write plaintext to directory/path
    end

    Sync->>Refs: write_head(commit_id)
    Sync-->>CLI: directory path
    CLI-->>User: "Cloned vault to <path>"
```

## Data Flow: Push Operation

```mermaid
sequenceDiagram
    participant User
    participant CLI as CLI__Vault
    participant Sync as Vault__Sync
    participant API as Vault__API
    participant Crypto as Vault__Crypto
    participant Store as Object_Store
    participant Refs as Ref_Manager

    User->>CLI: sg-send push
    CLI->>Sync: push(directory)
    Sync->>Sync: _read_vault_key(directory)
    Sync->>Crypto: derive_keys(passphrase, vault_id)
    Sync->>Sync: status(directory)
    Note over Sync: Compare local files vs tree in object store

    loop For each new/modified file
        Sync->>Sync: read file content
        Sync->>Crypto: encrypt(read_key, content)
        Crypto-->>Sync: ciphertext
        Sync->>Store: store(ciphertext)
        Store-->>Sync: object_id
        Sync->>API: write(vault_id, object_id, write_key, ciphertext)
    end

    Sync->>Sync: _build_tree_json(file_map)
    Sync->>Crypto: encrypt(read_key, tree_json)
    Sync->>API: write(vault_id, tree_file_id, write_key, encrypted_tree)

    Sync->>Sync: Build commit object (parent=HEAD, tree_id)
    Sync->>Crypto: encrypt(read_key, commit_json)
    Sync->>Store: store(encrypted_commit)

    Sync->>Sync: Build ref object (commit_id, version)
    Sync->>Crypto: encrypt(read_key, ref_json)
    Sync->>API: write(vault_id, ref_file_id, write_key, encrypted_ref)

    Sync->>Refs: write_head(new_commit_id)
    Sync-->>CLI: {pushed: [...], commit_id: "..."}
    CLI-->>User: "Pushed N files"
```

## Object Store Layout

```
.sg_vault/
├── VAULT-KEY                    # plaintext vault key (not in bare mode)
├── refs/
│   └── head                     # current commit ID (12 hex chars)
├── objects/
│   ├── a1/
│   │   └── b2c3d4e5f6          # encrypted blob (content-addressed)
│   ├── ff/
│   │   └── 0123456789          # encrypted blob
│   └── ...
└── (no tree.json — legacy format)

File Naming: objects/{id[0:2]}/{id[2:12]}
ID Computation: SHA256(ciphertext)[0:12]
```
