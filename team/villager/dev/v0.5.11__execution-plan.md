## Commit Strategy

Each task gets its own commit with format:
```
villager: <phase>/<sprint>/<task> — <description>

Example:
villager: phase-1/sprint-1/task-1.1 — add Schema__PKI_Key_Pair tests
villager: phase-1/sprint-2/task-2.1 — expand safe type boundary tests
villager: phase-2/sprint-5/task-5.1 — create Safe_UInt__Key_Size type
```

Run full test suite before each commit:
```bash
python -m pytest -q
```

## Risk Register

| Risk | Impact | Mitigation |
|------|--------|------------|
| New tests discover bugs | Low — document, don't fix | Write passing test for current behavior |
| Schema negative tests fail unexpectedly | Medium | Document as bug, test passes with current behavior |
| RF-2 (raw primitives) breaks PKI tests | High | Defer to Phase Three if tests break |
| Coverage target not reached | Low | Prioritize high-impact WPs first |
| Test performance degradation | Medium | Use setUpClass, avoid sleep, cap at 100ms/test |
