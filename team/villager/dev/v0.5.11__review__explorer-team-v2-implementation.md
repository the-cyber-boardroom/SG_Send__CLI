# Review: Explorer Team v2 Architecture Implementation

**Date:** 2026-03-14
**Reviewer:** Villager Team (Dev)
**Scope:** Explorer team's vault v2 merge ‚Äî 91 files, ~5,900 lines added
**Baseline:** 874 passed, 28 skipped | 89% overall coverage

---

## Executive Summary

The explorer team delivered a substantial vault v2 implementation: branching model, three-way merge, fetch/push with delta uploads, key management, and 12 new safe types with 10 new schemas. The work is solid ‚Äî 89% coverage, clean Type_Safe patterns in schemas, and well-structured manager classes. Three areas need attention: (1) raw primitives in operational classes, (2) coverage gaps in API/CLI layers, and (3) code duplication in Vault__Sync.

### Coverage by Module

| `api/` | **26-36%** | **Critical gap** |

### Critical Coverage Gaps

1. **`API__Transfer.py` (36%)** ‚Äî HTTP client methods almost entirely untested. Only envelope construction covered.
2. **`Vault__API.py` (26%)** ‚Äî Real implementation bypassed by `Vault__API__In_Memory` stub in tests.
3. **`CLI__Vault.py` (59%)** ‚Äî Many command handlers (init, commit, push, pull, log, diff) lack CLI-level tests.

### Moderate Gaps

4. **`Vault__Fetch.py` (82%)** ‚Äî Remote fetch code paths (lines 30-38) untested; only local LCA/chain walking covered.
5. **`Vault__Merge.py` (87%)** ‚Äî Conflict-file writing and merge-commit creation paths uncovered.

### Skipped Tests (28)

All 28 are integration tests ‚Äî appropriate skip conditions:
- 14 in `test_Vault__Integration.py` ‚Äî requires `SG_VAULT_KEY` env var
- 14 in `test_Vault__Local_Server.py` ‚Äî requires Python 3.12 venv with server package

### Test Quality Assessment

**Strengths:**
- Safe_type tests cover valid inputs, boundary values, regex violations, case normalization, and type preservation
- Schema tests verify round-trip invariant (`from_json(obj.json()).json() == obj.json()`)
- Sync tests exercise real multi-step workflows (init ‚Üí add ‚Üí commit ‚Üí push ‚Üí pull)
- Merge tests cover all 3-way scenarios: adds, modifies, deletes, conflicts, both-add-same, both-delete
- No mocks in explorer team's tests ‚Äî real objects, real crypto, real filesystem via `tempfile`

**Weaknesses:**
- `Vault__API__In_Memory` is copy-pasted across **8 test files** with minor variations ‚Äî maintenance risk
- No error-path tests for sync: corrupt data, key mismatches, partial failures, network errors
- Some schema tests only check constructor defaults (shape, not behavior)

### Scaffolding Recommendations

1. **Extract `Vault__API__In_Memory`** to shared `tests/conftest.py` ‚Äî eliminate 8 copies
2. **Add API-layer tests** ‚Äî even request construction and error handling without a live server
3. **Add CLI__Vault tests** ‚Äî argument parsing and output formatting for uncovered commands
4. **Add sync error-path tests** ‚Äî corrupt objects, key mismatches, partial push failures

## Phase 3: Safe Refactoring Opportunities

### Safe Refactors (Won't Break Tests)

| # | Refactor | File(s) | Lines Saved | Effort |
|---|----------|---------|-------------|--------|
| 1 | **Constants consolidation** ‚Äî `SG_VAULT_DIR` defined in 5 files, move to `constants.py` | 5 files | ~5 | Trivial |
| 2 | **Remove unused imports** ‚Äî `Schema__Object_Commit`, `Schema__Object_Tree` in Fetch.py | `Vault__Fetch.py` | 2 | Trivial |
| 3 | **Extract path extraction** ‚Äî `str(entry.path) if entry.path else str(entry.name)` repeated 7x | `Vault__Sync.py`, `Vault__Merge.py` | ~20 | Low |
| 4 | **Extract tree-to-map** ‚Äî tree entries ‚Üí dict conversion repeated 4x | `Vault__Sync.py`, `Vault__Merge.py` | ~30 | Low |
| 5 | **Extract conflict-file walker** ‚Äî identical `os.walk()` + filter in 2 methods | `Vault__Merge.py` | ~15 | Low |
| 6 | **Extract init helper** ‚Äî 7 methods repeat same component initialization (~10 lines each) | `Vault__Sync.py` | ~70 | Medium |
| 7 | **Extract long methods** ‚Äî `pull()` 129 lines, `push()` 106 lines, `commit()` 82 lines | `Vault__Sync.py` | ~150 | High |

### Risky Refactors (Would Need Test Updates)

| # | Refactor | Risk | Decision |
|---|----------|------|----------|
| 1 | Replace 16 `return dict(...)` with Type_Safe schemas | Tests check dict keys directly | **Defer to Phase 4** |
| 2 | Make `Vault__Storage.vault_path` required | ~60+ instantiation sites | **Defer** |
| 3 | Change `Vault__Fetch.find_lca()` return type | Callers depend on None handling | **Defer** |

### Patterns to Preserve

- Schema separation (data in `schemas/`, behavior in managers)
- Explicit dependency injection (crypto, api, storage as params)
- Private helpers with underscore prefix
- Three-way merge with case-by-case documentation
- LCA algorithm with parallel queue traversal
