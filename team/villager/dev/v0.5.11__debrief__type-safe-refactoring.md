## What Was Learned

1. **Safe_UInt default of 0 is compatible.** Replacing `int` with `Safe_UInt__Key_Size` preserves the default `== 0` behavior, so existing tests needed no changes.

2. **Safe_Str default of `''` is compatible.** Replacing `str` with `Safe_Str__PEM_Key` preserves the default `== ''` behavior. The `allow_empty = True` flag was essential.

3. **Semantic type mismatches are silent bugs.** `content_type: Safe_Str__File_Path` would have silently rejected valid MIME types containing `+` (e.g. `application/vnd.api+json`). The `Safe_Str__Content_Type` regex now correctly allows `+`.

4. **`Safe_Str__Base_URL` already existed.** The `endpoint_url` fix only required swapping the import â€” no new type needed. The URL regex correctly allows `:` and `/` which `Safe_Str__File_Path` also happened to allow, so this was purely a semantic correctness fix.

5. **Inline imports were widespread in CLI__PKI.** 8 inline `import os`/`import base64` statements across 6 methods. Moving them to top-level was safe because `os` and `base64` are stdlib modules with zero side effects on import.

6. **Round-trip invariant held throughout.** All schema round-trip tests (`from_json(obj.json()).json() == obj.json()`) passed without modification, confirming Type_Safe handles the new types transparently.
