## 1. Source Code Improvements

### `Vault__Sync.py` ‚Äî `_init_components()` extraction (biggest change)

**Problem:** Six methods (`commit`, `status`, `pull`, `push`, `merge_abort`, `branches`) each contained an identical ~10-line block that loaded the vault key, derived crypto keys, and instantiated `Vault__Storage`, `Vault__Object_Store`, `Vault__Ref_Manager`, `Vault__Key_Manager`, and `Vault__Branch_Manager`.

**Fix:** Extracted `_init_components(directory)` that returns a named dict with all initialized components. Each method now starts with a single call:

```python
c = self._init_components(directory)
```

This eliminated ~60 lines of duplication and makes adding new methods (e.g., future `clone()`) trivial.

### `Vault__Sync.py` ‚Äî `_entry_path()` extraction

**Problem:** The pattern `str(entry.path) if entry.path else str(entry.name)` appeared in 5+ locations.

**Fix:** Extracted `_entry_path(entry)` static-style helper.

### DRY constant consolidation

**Problem:** `SG_VAULT_DIR = '.sg_vault'` and `VAULT_KEY_FILE = 'VAULT-KEY'` were defined independently in `Vault__Storage`, `Vault__Inspector`, `Vault__Bare`, and `Vault__Legacy_Guard`.

**Fix:** All files now import from `Vault__Storage` as the single source of truth.

**Files changed:**
- `Vault__Inspector.py` ‚Äî import from `Vault__Storage`
- `Vault__Bare.py` ‚Äî import from `Vault__Storage`
- `Vault__Legacy_Guard.py` ‚Äî import from `Vault__Storage`

### Type annotation improvements

Replaced bare `list` and `dict` with parameterized versions across:
- `Vault__Fetch.py` ‚Äî `list` ‚Üí `list[str]`, removed unused imports
- `Vault__Merge.py` ‚Äî `list` ‚Üí `list[str]`, `dict` ‚Üí `dict[str, ...]`
- `Vault__Object_Store.py` ‚Äî `list` ‚Üí `list[str]`
- `Vault__Ref_Manager.py` ‚Äî `list` ‚Üí `list[str]`
- `Vault__Key_Manager.py` ‚Äî `list` ‚Üí `list[str]`

### Removed vestigial `Safe_Str__Batch_Op`

`Safe_Str__Batch_Op.py` was created early in our work before the Explorer team introduced `Enum__Batch_Op`. After the Phase C merge, `Schema__Batch_Operation` uses `Enum__Batch_Op`, making our safe-type vestigial. Removed both the type and its test.

## 2. Shared Test Infrastructure

### `tests/conftest.py` ‚Äî `Vault__API__In_Memory`

**Problem:** Multiple test files defined their own local `Vault__API__In_Memory` classes, leading to drift and missing methods. The Explorer team's Phase C batch API added `batch()` and `list_files()` to `Vault__API`, but test stubs didn't have them.

**Fix:** Created a single shared `Vault__API__In_Memory` in `tests/conftest.py` with:
- `write()`, `read()`, `delete()` ‚Äî basic CRUD
- `batch()` ‚Äî supports `write`, `write-if-match` (with CAS hash check), and `delete` operations
- `list_files()` ‚Äî prefix-filtered listing
- `_write_count` / `_batch_count` ‚Äî counters for test assertions

All test files now import from conftest instead of defining their own stubs. This includes the unit tests for Push V2, Pull V2, Init V2, Commit V2, and Helpers.
