### Detailed Findings on Key Files

**Best tests (exemplary quality):**

- `test_Vault__Crypto__Hardening.py` — 8 tests covering encrypt/decrypt round-trip, cross-key failure, nonce uniqueness, 1MB payload, empty payload, key derivation determinism, different salts, tampered ciphertext. All real objects. Comprehensive edge cases.

- `test_PKI__Crypto__Hardening.py` — 7 tests covering key generation, sign/verify round-trip, wrong-key failure, signature uniqueness, PEM serialization, public key extraction, empty data signing. All real objects.

- `test_AppSec__Vault_Security.py` — 6 test classes covering: no plaintext in object store, vault key not stored as object, secrets encrypted at rest, key derivation constants match spec, commit metadata has no sensitive data, tree structure encrypted. Uses real `Vault__API__In_Memory` subclass with actual storage. Exemplary "no mocks" pattern.

- `test_Safe_Str__File_Path.py` — Path traversal prevention (`../` rejection) is a security-relevant test. Good boundary coverage.

**Tests that will need updates after Phase A:**

- `test_Vault__Sync__Helpers.py` — Tests for `_flatten_tree()` and `_build_tree_json()` will be obsolete when sub-tree model replaces flat trees. Mark as "pre-architecture" tests.

- `test_Vault__Inspector__Coverage.py` — Inspector output will change with bare/ layout (branches, refs, data folders instead of objects/).

- `test_Schema__Negative_Tests.py` — Will need updates for `parent` → `parents[]` field change.

### Design Pattern Worth Noting

The `Vault__API__In_Memory` pattern in `test_AppSec__Vault_Security.py` (lines 16-33) is the correct way to create test doubles in this project:

```python
class Vault__API__In_Memory(Vault__API):
    def setup(self):
        self._store = {}
        return self

    def write(self, vault_id, file_id, write_key, payload):
        self._store[f'{vault_id}/{file_id}'] = payload
        return {'status': 'ok'}
    # ... real read/delete methods
```

This is a real implementation with actual behavior — not a mock. Other test files that need API test doubles should follow this pattern.
