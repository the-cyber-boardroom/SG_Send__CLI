## Integration Testing Infrastructure

A Python 3.12 venv is required for integration tests because `sgraph-ai-app-send` needs Python >= 3.12 while the default environment uses 3.11.

```bash
# Setup
python3.12 -m venv /tmp/sg-send-venv-312
/tmp/sg-send-venv-312/bin/pip install -e ".[dev]"
/tmp/sg-send-venv-312/bin/pip install sgraph-ai-app-send

# Run integration tests
/tmp/sg-send-venv-312/bin/python -m pytest tests/integration/ -v
```

The in-memory test server (`setup__send_user_lambda__test_server()`) provides a real HTTP FastAPI server with in-memory S3 storage — no external dependencies needed.

See `team/explorer/dev/python-3.12-venv-integration-testing.md` for full details.

## Architecture Quality Notes

### Strengths
- **Type_Safe compliance** — All new types follow the Safe_* pattern (no raw primitives)
- **Backward compatibility** — Push falls back to individual writes when batch unavailable
- **Test coverage** — Every new class has corresponding unit tests + integration tests
- **Zero breaking changes** — All pre-existing tests still pass
- **Crypto verification** — GC drain verifies ECDSA signatures before integrating change packs
- **CAS protection** — Push uses `write-if-match` to prevent lost updates

### Design debt
- Backend abstraction exists but isn't used by the sync engine (push/pull still use Vault__API directly)
- No zip backend for portable vault export
