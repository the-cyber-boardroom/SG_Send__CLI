## Phase 3: PKI Operations

**Complexity:** Large | **Ships in two parts: keygen+keyring first, then sign/verify/encrypt**

### New files

| File | Purpose |
|------|---------|
| `sg_send_cli/safe_types/Safe_Str__Public_Key.py` | PEM/base64 public key validation |
| `sg_send_cli/safe_types/Safe_Str__Private_Key.py` | PEM private key validation |
| `sg_send_cli/safe_types/Safe_Str__Key_Fingerprint.py` | SHA-256 fingerprint of public key |
| `sg_send_cli/safe_types/Safe_Str__Signature.py` | Base64 signature validation |
| `sg_send_cli/schemas/Schema__PKI_Key_Pair.py` | public_key, private_key, fingerprint, created_at |
| `sg_send_cli/schemas/Schema__PKI_Public_Key.py` | public_key, fingerprint, label |
| `sg_send_cli/crypto/PKI__Crypto.py` | generate_key_pair, sign, verify, encrypt_for_recipient, decrypt_as_recipient |
| `sg_send_cli/pki/PKI__Keyring.py` | Public key store at `~/.sg-send/keyring.json` |
| `sg_send_cli/pki/PKI__Key_Store.py` | Private keys encrypted at `~/.sg-send/keys/` |

### Algorithm decisions (must lock down before implementation)

- **Signing:** ECDSA P-256 (Web Crypto compatible) or Ed25519 (not in Web Crypto as of 2025)
- **Encryption:** ECDH P-256 + HKDF + AES-256-GCM (ECIES, Web Crypto compatible)
- **Key format:** JWK (Web Crypto native) or PEM
- **Test vectors are mandatory** per crypto interop requirement

### Tests

| File | What it tests |
|------|---------------|
| `tests/unit/crypto/test_PKI__Crypto.py` | Generate, sign/verify round-trip, encrypt/decrypt round-trip, test vectors |
| `tests/unit/pki/test_PKI__Keyring.py` | Add/list/remove keys |
| `tests/unit/pki/test_PKI__Key_Store.py` | Store/load encrypted private keys |

## Phase 4: Secrets Management

**Complexity:** Medium | **Ships independently after Phase 0**

### Approach
Encrypted-file store using existing `Vault__Crypto` (no new dependencies). Secrets stored as AES-256-GCM encrypted JSON at `~/.sg-send/secrets.enc`. Master key derived from user passphrase via PBKDF2.

### New files

| File | Purpose |
|------|---------|
| `sg_send_cli/safe_types/Safe_Str__Secret_Key.py` | Key name validation |
| `sg_send_cli/schemas/Schema__Secret_Entry.py` | key, created_at |
| `sg_send_cli/secrets/Secrets__Store.py` | CRUD: store, get, list_keys, delete |

### Tests

| File | What it tests |
|------|---------------|
| `tests/unit/secrets/test_Secrets__Store.py` | CRUD with temp encrypted file, encryption round-trip |
