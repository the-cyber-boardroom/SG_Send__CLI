## Summary

The push/pull workflows and three-way merge design are **architecturally sound** (8/10). The conflict-on-clone-branch innovation is excellent. Seven specific gaps identified in merge edge cases, retry strategy, and enforcement. Specification completeness is 6/10 — core concepts are clear but edge cases need decisions before Phase B implementation.

---

## Critical Gaps

### 1. Push Step 4→6 Merge Ordering (Medium Risk)

Step 4 merges current INTO clone (may conflict). Step 6 merges clone INTO current (must always be clean). This is correct IF:
- Step 6 is never reached when step 4 produced conflicts
- Step 6 uses the correct merge base

**Implementation requirement:** Add strict pre-condition check before step 6. If any `.conflict` files exist, step 6 must not execute.

### 2. File Moves/Renames in Three-Way Merge (Medium Risk)

The sub-tree shortcut compares `tree_id` values. When a file is moved between directories:
- Old tree loses the entry, new tree gains it → both `tree_id`s change
- Algorithm sees "delete + add" instead of "move"
- May produce false conflict

**Decision needed:** Accept false conflicts for moves in v1 (document as limitation) or add move detection using `blob_id` matching across directories.

### 3. LCA Strategy for Criss-Cross Merges (Medium Risk)

Spec says "find a shared commit" but doesn't specify which one if multiple LCAs exist. Options:
- Most recent by timestamp (simple, may be wrong if clocks drift)
- Recursive merge of all LCAs (complex, git's approach)
- First found (BFS order, depends on traversal order)

**Decision needed:** Choose strategy and document.

### 4. .conflict File Convention Edge Cases (Low Risk)

Unspecified: extensionless files, existing `.conflict` files, binary files, directory-vs-file conflicts. See main review document for full table.

### 5. merge --abort GC Dependency (High Risk)

Orphaned commits accumulate until GC runs. GC is only triggered on push/pull/merge. Local-only users who abort merges accumulate orphans forever.

**Recommendation:** Add `sg-send-cli gc` command or trigger GC on every commit.

### 6. Fast-Forward Enforcement (Low Risk)

Nothing prevents buggy code from fast-forwarding in wrong places. Add explicit ancestor check before any fast-forward.

### 7. Push Retry Strategy (Medium Risk)

No retry limit, no backoff. Risk of livelock with concurrent pushers.

**Recommendation:** Max 5 retries, exponential backoff (500ms → 8s) with jitter.

## What's Excellent

1. **Conflict isolation to clone branch** — prevents blocking other users. Best design decision in the spec.
2. **Fetch-first-then-merge push** — eliminates silent overwrites (most dangerous current bug).
3. **content_hash** — fixes Vault__Sync.py:359 size-only detection bug.
4. **Sub-tree merge shortcut** — O(changed directories) not O(total files). Identical sub-trees skipped entirely.
5. **Natural delta push** — only changed blobs + path trees uploaded. No diff algorithm needed.
