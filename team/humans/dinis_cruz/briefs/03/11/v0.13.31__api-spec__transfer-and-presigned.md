## `api/transfers` — Core Transfer Lifecycle

This is the standard (non-S3) path. Suitable for files under ~5MB. For larger files, use `api/presigned`.

---

### `POST /api/transfers/create`

Create a new transfer and get an upload URL.

**Auth:** Access token required (header, query param, or MCP tool param)  
**Token header:** `x-sgraph-access-token: <token>`  
**Query param fallback:** `?access_token=<token>`

**Request body:**
```json
{
  "file_size_bytes": 12345,
  "content_type_hint": "application/octet-stream"
}
```

**Response:**
```json
{
  "transfer_id": "a1b2c3d4e5f6",
  "upload_url": "/transfers/upload/a1b2c3d4e5f6"
}
```

**Notes:**
- `transfer_id` is a 12-char random hex string
- `content_type_hint` is a MIME type string

---

### `POST /api/transfers/upload/{transfer_id}`

Upload the encrypted payload for a transfer.

**Auth:** Access token required  
**Supports two upload modes:**

**Mode 1 — Raw bytes (browser/CLI):**  
Send encrypted bytes as raw request body with `Content-Type: application/octet-stream`

**Mode 2 — MCP tool parameter:**  
Send base64-encoded payload as `data` query parameter:
```
POST /api/transfers/upload/a1b2c3d4e5f6?data=<base64>&access_token=<token>
```

**Mode 3 — JSON-wrapped base64 (legacy MCP):**  
Send `{"data": "<base64>"}` as request body

**Response:**
```json
{
  "status": "uploaded",
  "transfer_id": "a1b2c3d4e5f6",
  "size": 12345
}
```

**Error:** `404` if transfer not found or not in `pending` state

---

### `POST /api/transfers/complete/{transfer_id}`

Mark a transfer as complete and get the download URL.

**Auth:** Access token required

**Response:**
```json
{
  "transfer_id": "a1b2c3d4e5f6",
  "download_url": "/d/a1b2c3d4e5f6",
  "transparency": {
    "ip": "<sha256-of-sender-ip>",
    "timestamp": "2026-03-11T10:00:00Z",
    "file_size_bytes": 12345,
    "stored_fields": ["ip_hash", "file_size_bytes", "created_at", "content_type_hint"],
    "encrypted": ["file_name", "file_content"],
    "not_stored": ["decryption_key", "raw_ip"]
  }
}
```

**Error:** `404` if transfer not found or payload not yet uploaded  
**Note:** Token name is intentionally NOT included in response (was leaking into shareable URLs — security fix)

---

### `GET /api/transfers/info/{transfer_id}`

Get transfer metadata without downloading the payload.

**Auth:** None required

**Response:**
```json
{
  "transfer_id": "a1b2c3d4e5f6",
  "status": "completed",
  "file_size_bytes": 12345,
  "content_type_hint": "application/octet-stream",
  "created_at": "2026-03-11T10:00:00Z",
  "download_count": 2
}
```

**Status values:** `pending`, `completed`  
**Error:** `404` if not found

---

### `GET /api/transfers/download/{transfer_id}`

Download the raw encrypted payload bytes.

**Auth:** None required  
**Size limit:** 5MB (`LAMBDA_RESPONSE_LIMIT`). Returns `413` with redirect hint for larger files.

**Response:** Raw bytes, `Content-Type: application/octet-stream`

**Errors:**
- `404` — transfer not found
- `413` — file too large, use `/presigned/download-url/{transfer_id}` instead

---

### `GET /api/transfers/download-base64/{transfer_id}`

Download the encrypted payload as a base64-encoded JSON response (MCP-safe).

**Auth:** None required  
**Size limit:** ~3.75MB (`LAMBDA_BASE64_LIMIT`). Returns `413` with redirect hint for larger files.

**Response:**
```json
{
  "transfer_id": "a1b2c3d4e5f6",
  "data": "<base64-encoded-bytes>",
  "file_size_bytes": 12345
}
```

---

### `GET /api/transfers/check-token/{token_name}`

Check if a token exists and is active. Does **not** consume a use.

**Auth:** None required

**Response:**
```json
{
  "valid": true,
  "status": "active",
  "remaining": 42
}
```

```json
{
  "valid": false,
  "reason": "not_found"
}
```

**Dev mode (no admin service):** always returns `{"valid": true, "status": "active"}`

---

### `POST /api/transfers/validate-token/{token_name}`

Validate a token and **consume one use** (used when download page is opened).

**Auth:** None required

**Response:** (from admin service) — token usage recorded, IP hash logged

## `api/presigned` — Large File Transfer (S3 Presigned URLs)

Use this path for files over ~5MB. The server acts as a coordinator — it creates the transfer record and returns presigned S3 URLs. The actual upload/download goes directly to S3, bypassing Lambda size limits.

---

### `GET /api/presigned/capabilities`

Check which upload modes are available on this server instance.

**Auth:** None required

**Response:** (from `Service__Presigned_Urls.get_capabilities()`)
```json
{
  "presigned_available": true,
  "multipart_available": true,
  "max_part_size_bytes": 104857600,
  "min_part_size_bytes": 5242880
}
```

---

### `POST /api/presigned/initiate`

Initiate a multipart upload. Returns presigned S3 URLs for each part.

**Auth:** Access token required (header only — no query param fallback)  
**Token header:** `x-sgraph-access-token: <token>`

**Request body:**
```json
{
  "transfer_id": "a1b2c3d4e5f6",
  "file_size_bytes": 104857600,
  "num_parts": 10
}
```

**Response:** (from `Service__Presigned_Urls.initiate_multipart_upload`)
```json
{
  "upload_id": "<s3-multipart-upload-id>",
  "parts": [
    { "part_number": 1, "upload_url": "https://s3.amazonaws.com/..." },
    { "part_number": 2, "upload_url": "https://s3.amazonaws.com/..." }
  ]
}
```

**Errors:**
- `404` — transfer not found
- `409` — transfer not in pending state
- `400` — presigned URLs not available on this instance, or too many parts

---

### `POST /api/presigned/complete`

Complete a multipart upload by providing the ETags for each part.

**Auth:** Access token required

**Request body:**
```json
{
  "transfer_id": "a1b2c3d4e5f6",
  "upload_id": "<s3-multipart-upload-id>",
  "parts": [
    { "part_number": 1, "etag": "\"abc123\"" },
    { "part_number": 2, "etag": "\"def456\"" }
  ]
}
```

**Response:** success confirmation from S3 complete operation

---

### `POST /api/presigned/abort/{transfer_id}/{upload_id}`

Abort a multipart upload in progress. Cleans up partial S3 parts.

**Auth:** Access token required

---

### `GET /api/presigned/upload-url/{transfer_id}`

Get a single presigned S3 PUT URL for small-to-medium files (< 5GB, single part).

**Auth:** Access token required

**Response:**
```json
{
  "upload_url": "https://s3.amazonaws.com/...?X-Amz-Signature=..."
}
```

---

### `GET /api/presigned/download-url/{transfer_id}`

Get a presigned S3 GET URL for downloading. No size limit — stream directly from S3.

**Auth:** None required (download is open once transfer is completed)

**Response:**
```json
{
  "download_url": "https://s3.amazonaws.com/...?X-Amz-Signature=..."
}
```

**Errors:**
- `404` — transfer not found
- `409` — transfer not yet completed

## Client-Side Encryption Protocol

**This is mandatory.** The server stores and serves only opaque encrypted bytes. It has no knowledge of filenames, file content, or decryption keys. All crypto happens on the client before upload and after download.

---

### Step 1 — SGMETA Envelope (files only, not text)

Before encrypting a file, the client wraps it in the **SGMETA envelope** — a simple binary format that preserves the original filename (and any other metadata) inside the encrypted blob.

**Binary layout:**

```
[ MAGIC (7 bytes) ][ META_LEN (4 bytes, big-endian uint32) ][ META_JSON (META_LEN bytes) ][ FILE_CONTENT (remaining bytes) ]
```

| Field | Bytes | Value |
|---|---|---|
| Magic | 7 | `0x53 0x47 0x4D 0x45 0x54 0x41 0x00` (ASCII: `SGMETA\0`) |
| Meta length | 4 | Big-endian uint32, length of the JSON metadata block |
| Meta JSON | variable | UTF-8 JSON, currently `{"filename": "original_name.ext"}` |
| File content | remaining | Raw file bytes |

**Python implementation:**

```python
import json, struct

SGMETA_MAGIC = bytes([0x53, 0x47, 0x4D, 0x45, 0x54, 0x41, 0x00])

def package_with_metadata(content_bytes: bytes, filename: str) -> bytes:
    meta_bytes = json.dumps({"filename": filename}).encode('utf-8')
    meta_len   = struct.pack('>I', len(meta_bytes))          # 4-byte big-endian
    return SGMETA_MAGIC + meta_len + meta_bytes + content_bytes

def unpackage_metadata(data: bytes) -> tuple[dict | None, bytes]:
    magic = SGMETA_MAGIC
    if len(data) < len(magic) + 4:
        return None, data
    if data[:len(magic)] != magic:
        return None, data                                    # No envelope — treat as raw bytes
    meta_len   = struct.unpack('>I', data[len(magic):len(magic)+4])[0]
    meta_start = len(magic) + 4
    content_start = meta_start + meta_len
    if content_start > len(data):
        return None, data
    try:
        metadata = json.loads(data[meta_start:content_start].decode('utf-8'))
        return metadata, data[content_start:]
    except Exception:
        return None, data
```

**Text mode** (when uploading plain text rather than a file) skips the SGMETA envelope entirely — the plaintext is encoded directly to UTF-8 bytes and encrypted as-is.

---

### Step 2 — AES-256-GCM Encryption

**Algorithm:** AES-256-GCM (via Web Crypto API in browser, or Python `cryptography` library in CLI)  
**Key length:** 256-bit  
**IV length:** 12 bytes (96 bits), randomly generated per transfer  
**Key export format:** Raw bytes, base64url-encoded (no padding, `+` → `-`, `/` → `_`)

**Encrypted payload layout:**

```
[ IV (12 bytes) ][ CIPHERTEXT + GCM_AUTH_TAG (remaining bytes) ]
```

The GCM authentication tag (16 bytes) is appended to the ciphertext by the crypto library automatically — no separate handling needed.

**Python implementation:**

```python
import os, base64
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

def generate_key() -> bytes:
    return os.urandom(32)                                    # 256-bit key

def export_key(key: bytes) -> str:
    return base64.urlsafe_b64encode(key).rstrip(b'=').decode()

def import_key(key_str: str) -> bytes:
    padding = 4 - (len(key_str) % 4)
    key_b64 = key_str.replace('-', '+').replace('_', '/') + ('=' * (padding % 4))
    return base64.b64decode(key_b64)

def encrypt(key: bytes, plaintext: bytes) -> bytes:
    iv         = os.urandom(12)
    aesgcm     = AESGCM(key)
    ciphertext = aesgcm.encrypt(iv, plaintext, None)         # None = no AAD
    return iv + ciphertext                                   # IV prepended

def decrypt(key: bytes, encrypted: bytes) -> bytes:
    iv         = encrypted[:12]
    ciphertext = encrypted[12:]
    aesgcm     = AESGCM(key)
    return aesgcm.decrypt(iv, ciphertext, None)
```

---

### Step 3 — Key in URL Fragment

The decryption key is **never sent to the server**. It is passed to the recipient via the URL hash fragment, which browsers do not include in HTTP requests:

```
https://send.sgraph.ai/en-gb/download/#<transfer_id>/<key_base64url>
```

Example:
```
https://send.sgraph.ai/en-gb/download/#a1b2c3d4e5f6/xK8mP2vN9...
```

For split-channel security (send link and key separately), the UI also supports:
- **Link only:** `https://send.sgraph.ai/en-gb/download/#<transfer_id>`
- **Key separately:** `<key_base64url>` (share via different channel)

### Full Upload Sequence (CLI)

```python
import os, json, struct, base64
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import requests

BASE_URL = "https://send.sgraph.ai"
TOKEN    = "your-access-token"

def upload_file(filepath: str) -> str:
    filename      = os.path.basename(filepath)
    content_bytes = open(filepath, 'rb').read()

    # 1. Wrap in SGMETA envelope (preserves filename)
    meta_bytes    = json.dumps({"filename": filename}).encode()
    plaintext     = SGMETA_MAGIC + struct.pack('>I', len(meta_bytes)) + meta_bytes + content_bytes

    # 2. Generate key + encrypt
    key           = os.urandom(32)
    iv            = os.urandom(12)
    ciphertext    = AESGCM(key).encrypt(iv, plaintext, None)
    encrypted     = iv + ciphertext

    # 3. Create transfer
    resp          = requests.post(f"{BASE_URL}/api/transfers/create",
                        json={"file_size_bytes": len(encrypted),
                              "content_type_hint": "application/octet-stream"},
                        headers={"x-sgraph-access-token": TOKEN})
    transfer_id   = resp.json()["transfer_id"]

    # 4. Upload encrypted payload
    requests.post(f"{BASE_URL}/api/transfers/upload/{transfer_id}",
                  data=encrypted,
                  headers={"x-sgraph-access-token": TOKEN,
                           "Content-Type": "application/octet-stream"})

    # 5. Complete
    requests.post(f"{BASE_URL}/api/transfers/complete/{transfer_id}",
                  headers={"x-sgraph-access-token": TOKEN})

    # 6. Build shareable URL (key in fragment, never sent to server)
    key_str = base64.urlsafe_b64encode(key).rstrip(b'=').decode()
    return f"{BASE_URL}/en-gb/download/#{transfer_id}/{key_str}"
```

### Full Download Sequence (CLI)

```python
def download_file(transfer_id: str, key_str: str, out_path: str):
    # 1. Download encrypted payload
    resp      = requests.get(f"{BASE_URL}/api/transfers/download/{transfer_id}")
    encrypted = resp.content

    # 2. Decrypt
    padding   = 4 - (len(key_str) % 4)
    key       = base64.b64decode(key_str.replace('-','+').replace('_','/') + '='*(padding%4))
    iv        = encrypted[:12]
    plaintext = AESGCM(key).decrypt(iv, encrypted[12:], None)

    # 3. Unpack SGMETA envelope to recover filename
    metadata, content = unpackage_metadata(plaintext)
    filename  = (metadata or {}).get("filename") or os.path.basename(out_path)

    # 4. Save
    with open(filename, 'wb') as f:
        f.write(content)
    return filename
```

## CLI Implementation Guide (Phase 1)

### Upload workflow (small files, < 5MB)

```python
# 1. Create transfer
resp = POST /api/transfers/create
       body: {file_size_bytes, content_type_hint}
       headers: {x-sgraph-access-token: <token>}
→ {transfer_id, upload_url}

# 2. Upload payload
resp = POST /api/transfers/upload/{transfer_id}
       body: <raw encrypted bytes>
       headers: {x-sgraph-access-token: <token>, Content-Type: application/octet-stream}
→ {status: "uploaded", transfer_id, size}

# 3. Complete
resp = POST /api/transfers/complete/{transfer_id}
       headers: {x-sgraph-access-token: <token>}
→ {transfer_id, download_url, transparency}
```

### Upload workflow (large files, > 5MB)

```python
# 1. Create transfer (same as above)

# 2. Check capabilities
resp = GET /api/presigned/capabilities
→ {presigned_available, multipart_available, ...}

# 3. Initiate multipart
resp = POST /api/presigned/initiate
       body: {transfer_id, file_size_bytes, num_parts}
→ {upload_id, parts: [{part_number, upload_url}, ...]}

# 4. Upload each part directly to S3 (PUT to presigned URL)
for part in parts:
    PUT <upload_url>  body: <part bytes>
    → collect ETag from response header

# 5. Complete multipart
resp = POST /api/presigned/complete
       body: {transfer_id, upload_id, parts: [{part_number, etag}, ...]}

# 6. Complete transfer (same as above)
```

### Download workflow

```python
# Small file (< 5MB):
payload_bytes = GET /api/transfers/download/{transfer_id}

# MCP/JSON-safe (< 3.75MB):
resp = GET /api/transfers/download-base64/{transfer_id}
payload_bytes = base64.decode(resp["data"])

# Large file (any size):
resp = GET /api/presigned/download-url/{transfer_id}
payload_bytes = GET <resp["download_url"]>   # direct S3, no auth needed
```

### Suggested CLI commands

```bash
sg-send-cli upload <file> [--token <token>]           # auto-selects small/large path
sg-send-cli upload <file> --zip [--token <token>]     # zip before upload
sg-send-cli download <transfer-id> [--out <file>]     # auto-selects download path
sg-send-cli download <transfer-id> --base64           # force base64 (MCP mode)

```
