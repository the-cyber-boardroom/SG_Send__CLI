## Goal

Build the foundational types, schemas, and object store infrastructure for the git-like encrypted object store. No existing sync logic changes yet (that's Phase 2).

## Scope

### New Safe Types (3)
- `Safe_Str__Object_Id` ‚Äî 12-char hex, content-addressed (distinct from random `Safe_Str__File_Id`)
- `Safe_Str__Commit_Message` ‚Äî max 500 chars, printable + whitespace
- `Safe_Str__ISO_Timestamp` ‚Äî ISO 8601 UTC format (e.g., `2026-03-05T12:00:00.000Z`)

### New Schemas (4)
- `Schema__Object_Ref` ‚Äî HEAD pointer: `{commit_id, version}`
- `Schema__Object_Tree` ‚Äî flat tree: list of `{path, blob_id, size}` entries
- `Schema__Object_Tree_Entry` ‚Äî individual tree entry
- `Schema__Object_Commit` ‚Äî commit: `{parent, tree_id, timestamp, message, version}`

### Crypto Extensions (2 methods)
- `compute_object_id(ciphertext)` ‚Äî SHA-256 prefix of ciphertext ‚Üí 12-char hex
- `derive_ref_file_id(read_key, vault_id)` ‚Äî deterministic server slot for HEAD ref

### Object Store Module (3 classes)
- `Vault__Object_Store` ‚Äî local `.sg_vault/objects/` cache (git-style 2-char prefix dirs)
- `Vault__Ref_Manager` ‚Äî manages `.sg_vault/refs/head` file
- `Vault__Inspector` ‚Äî development tool for inspecting vault state, objects, trees, and commit chains

### Legacy Vault Protection
- `Vault__Legacy_Guard` ‚Äî detects old-format vaults and refuses to operate on them
- Guards added to `Vault__Sync.pull()`, `push()`, `status()`, `remote_status()`
- `clone()` is safe ‚Äî creates new-format vaults only

### CLI Inspect Commands (5 commands)
- `inspect` ‚Äî vault state overview
- `inspect-object <id>` ‚Äî object details + integrity check
- `inspect-tree` ‚Äî current tree entries
- `inspect-log` ‚Äî commit chain
- `inspect-stats` ‚Äî object store statistics

## Design Decisions

1. **Tree entries as list, not dict** ‚Äî consistent with existing `Schema__Vault_Index` pattern; each entry includes its `path` field
2. **Object_Id semantically distinct from File_Id** ‚Äî same 12-char hex format but different types to prevent mixing content-addressed IDs with random IDs
3. **Legacy detection: tree.json exists + no refs/head** ‚Äî simple, reliable, no false positives
4. **Inspector returns dicts** ‚Äî easy to serialize, test, and display; format methods produce human-readable output
